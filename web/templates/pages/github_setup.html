GitHub App Setup

Create a GitHub App for automatic deployments using Talos's manifest flow.

What this does

Talos sends a GitHub App manifest that preconfigures the webhook URL, callback URL, permissions, and subscribed events for you.

Talos URL

{{.TalosURL}}

Create GitHub App

What to expect on GitHub

  • 1. GitHub opens its standard GitHub App registration page.
  • 2. The app name may still be visibly editable. That is expected for the manifest flow.
  • 3. You do not need to manually fill in the webhook URL, callback URL, permissions, or subscribed events.
  • 4. After you create the app, GitHub redirects back to Talos and Talos stores the generated credentials automatically.

No Domain Yet?

{{if .HasDomain}}

Talos is configured with a domain, so the generated GitHub App URLs will use HTTPS for webhook delivery and setup callbacks.

{{else}}

Talos is currently generating GitHub App URLs from the current host, which may be an IP-based HTTP address. This is acceptable for testing, but a real domain with HTTPS is recommended before relying on GitHub App webhooks in production.

Once you configure TALOS_DOMAIN and point DNS to this server, rerun the setup so GitHub uses the final HTTPS URLs.
